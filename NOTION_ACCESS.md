# Підключення Notion до проекту

Цей файл — покрокова інструкція. Після налаштування доступ можна використовувати в n8n і в скриптах.

---

## Крок 1. Створити інтеграцію в Notion

1. Відкрий **https://www.notion.so/my-integrations**
2. Натисни **«+ New integration»**
3. Заповни:
   - **Name:** наприклад `Autopilot Content` або `CRMCUSTOMS`
   - **Associated workspace:** вибери свій робочий простір
   - **Capabilities:** увімкни **Read content**, **Update content**, **Insert content** (повний доступ на читання та запис)
4. Натисни **«Submit»**
5. У блоці **«Internal Integration Token»** натисни **«Show»** і **скопіюй токен** (починається з `secret_`). Збережи його — він буде лише раз.

---

## Крок 2. Створити «директорію» (батьківську сторінку)

1. У Notion створи **нову сторінку** (або використай існуючу).
2. Назва — наприклад **«Autopilot Content»** або **«CRMCUSTOMS — контент»**. Тут згодом будуть усі 9 баз (таблиці).
3. **Дати доступ інтеграції:**
   - Відкрий цю сторінку.
   - У правому верхньому куті натисни **«…»** (або **«Share»**).
   - Обери **«Add connections»** / **«Подключения»**.
   - Вибери свою інтеграцію (назву з кроку 1) і підтверди.

Тепер інтеграція має доступ до цієї сторінки та всього, що ти створиш всередині неї (підсторінки, таблиці).

---

## Крок 3. Дізнатися ID батьківської сторінки

ID сторінки потрібен, щоб створювати бази «всередині» неї через API.

1. Відкрий цю сторінку в браузері.
2. Подивись URL. Він виглядає так:
   - `https://www.notion.so/My-Page-**a1b2c3d4e5f6...**?v=...`
   - або `https://www.notion.so/**a1b2c3d4e5f6789012345678abcdef12**`
3. **ID сторінки** — це довгий рядок з букв і цифр (іноді з дефісами). Довжина 32 або 36 символів. Скопіюй його.

Якщо в URL є дефіси — при вставці в `.env` можна залишити як є або прибрати їх (Notion API приймає обидва варіанти).

---

## Крок 4. Зберегти токен і ID у проекті

1. У папці проекту («Контент менеджер») є файл **`.env.example`**.
2. **Створи копію** цього файлу і назви її **`.env`** (крапка на початку, без .example).
3. Відкрий **`.env`** у редакторі і заповни:
   ```
   NOTION_TOKEN=secret_твій_токен_тут
   NOTION_PARENT_PAGE_ID=id_сторінки_з_кроку_3
   ```
4. Збережи файл. Файл `.env` не потрапляє в git (додано в `.gitignore`) — токен ніхто не побачить у репозиторії.

**Не вставляй токен у чат** — лише в `.env`. Якщо вже вставив у чат — краще зміни токен у Notion (revoke старий і створи новий) і впиши новий у `.env`.

---

## Перевірка

Коли `.env` заповнено, можна перевірити доступ: у новому чаті попросити асистента «перевір підключення до Notion» — він прочитає `.env` (якщо ти дозволиш) і зможе викликати API (наприклад, отримати список дочірніх сторінок). Або перевірку зробимо в n8n пізніше.

---

## Файли

| Файл | Призначення |
|------|-------------|
| `.env` | Твій токен і ID сторінки (не комітити). Створюєш сам з .env.example. |
| `.env.example` | Приклад — які змінні потрібні. |
| `.gitignore` | Перелік файлів, які git ігнорує (тут — .env). |

*Після виконання кроків 1–4 доступ до Notion готовий для створення 9 баз.*
