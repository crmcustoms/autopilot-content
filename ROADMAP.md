## ЧЕРГА РЕАЛІЗАЦІЇ

### КРОК 1 — Planfix новини → Блог (РОБИМО ЗАРАЗ)
**Чому перше:** найпростіше — зображення беремо прямо з Planfix, не треба генерувати.

**Флоу:**
```
Planfix блог (RSS/scrape)
  → нова стаття знайдена
  → Claude: рерайт українською + генерація всіх SEO полів
  → Notion Blog DB: заповнити properties + записати blocks (тіло)
  → Status = "Публікація", Публиковать = true
  → Сайт автоматично підхоплює через n8n endpoint
  → (опційно) короткий TG пост → бот → схвалення → канал
```

**Що треба зробити технічно:**
- [x] Скрипт моніторингу Planfix блогу (RSS або scraping) → `scripts/planfix_to_blog.py`
- [x] Функція: Claude рерайтить статтю + генерує slug/H1/H2/H3/Title/desc/keywords
- [x] Функція: створення Notion сторінки з blocks (heading_2 + paragraph + image)
- [x] Функція: генерація TG анонсу → content_plan (На затвердженні)

**Запуск:**
- `run_planfix_monitor.bat` — моніторинг (до 3 нових статей)
- `run_planfix_one.bat <URL>` — тест однієї статті
- `python scripts/planfix_to_blog.py <URL>` — те саме через консоль

**Потрібно від Максима:**
1. ~~URL Planfix блогу або RSS feed~~ → заповнити `PLANFIX_BLOG_URL` в `.env`
2. URL n8n ендпоінту для тесту (опційно — перевірити що стаття з'явилась на сайті)

### КРОК 2 — Кейси → Cases DB (після Кроку 1)
**Флоу:**
```
Максим надсилає дані кейсу в бот (або заповнює Notion)
  → Claude: генерує структурований текст кейсу
  → Зображення: беремо з сайту клієнта АБО генеруємо через Replicate
  → Notion Cases DB: properties + blocks
  → Status = "Shared"

  → TG анонс → схвалення → канал
```

**Replicate для кейсів:**
Потрібна модель що генерує UI/дашборди/скріншоти інтерфейсів.

### КРОК 6 — Аудіокнижки (ВЕЛИКИЙ ПОТЕНЦІАЛ 🎧)
**Ідея (01.03.2026):** Публікація аудіокнижок на crmcustoms.com та у TG-каналі.
Аудіокнижки безкоштовно + від CRMCUSTOMS = великий трафік та довіра аудиторії.

**Флоу:**
```
Джерело: бізнес-книга (текст з відкритих джерел / саммарі)
  → Claude: генерує структурований саммарі/повний текст книги
  → ElevenLabs: озвучує текст (голос TTS)
  → Notion: нова сторінка у Blog DB (тип = audiobook)
  → Сайт: окрема сторінка з аудіоплеєром та текстом
  → TG канал: анонс + короткий уривок
  → (опційно) YouTube: відео з обкладинкою
```

**Джерела контенту:**
- Саммарі відомих бізнес-книг (Dale Carnegie, Seth Godin, Jim Collins тощо)
- Практичні книги по CRM, продажах, автоматизації — тематика сайту
- Книги у відкритому доступі (Project Gutenberg та ін.)

**Технічна реалізація:**
- ElevenLabs API або Google TTS для озвучення (є 5 GOOGLE_AI_KEY у .env — безкоштовно!)
- Аудіофайл → S3 (`crmcustoms.site`) → URL у поле `Фото 2 соц` або окреме поле
- Або: embed плеєр (SoundCloud, Spotify Podcast, anchor.fm) → Notion embed block
- Окремий скрипт: `scripts/audiobooks.py`

**Чому це великий потенціал:**
- Аудіокнижки безкоштовно — унікальна пропозиція для україномовної аудиторії
- SEO-трафік: "аудіокнижка [назва] українською"
- Retention: люди повертаються за новими книгами
- Позиціонування CRMCUSTOMS як лідера думок у бізнес-освіті

**Блокер:** Потрібна чіткість по:
- Якій тип плеєра/хостингу аудіо?
- Саммарі чи повний текст?
- Список перших 5 книг для старту?

**Статус: ЗАПЛАНОВАНИЙ (після Кроку 2-3)**
