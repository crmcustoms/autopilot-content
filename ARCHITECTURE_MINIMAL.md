# Процеси з ТЗ — що робити і без n8n

## Що ТЗ пропонувало через n8n

| Процес | Тригер | Що має відбуватися | Чи потрібен n8n |
|--------|--------|--------------------|------------------|
| **Бот: прийом команд** | Повідомлення в Telegram | Користувач пише /plan, /today, /strategy тощо — треба щось відповісти (дані з Notion або запустити генерацію). | **Ні** — достатньо бота (скрипт), який приймає updates і виконує логіку. |
| **Strategy runner** | Команда /strategy | Взяти промпти #1–#4 з Notion → послідовно викликати Claude → записати результат у бази Notion (Стратегія, Кути, План). | **Ні** — скрипт: читає промпти з Notion, викликає Claude API, пише в Notion. Бот викликає цей скрипт по /strategy. |
| **Затвердження (кнопки)** | Callback від кнопки в TG | Користувач натискає «Опублікувати» / «Відхилити» → оновити статус запису в Notion. | **Ні** — той самий бот: обробляє `callback_query`, робить PATCH сторінки в Notion. |
| **План на 7 днів / Сьогодні** | /plan, /today | Прочитати з Notion контент-план за датами, сформувати текст, відправити в чат. | **Ні** — бот: запит до Notion API (filter по даті), форматування, `sendMessage`. |
| **Генерація нового плану** | /regenerate або умова «залишилось &lt;10 постів» | Згенерувати план на 30 днів (Claude + промпт content_plan_gen) → записати в Notion, можливо повідомити в TG. | **Ні** — скрипт + Claude; бот по /regenerate викликає скрипт або напряму викликає API. |
| **Розклад публікацій (щодня о 08:00)** | Cron | Подивитися, що заплановано на сьогодні, при потребі щось запустити (наприклад, відправити в чергу на публікацію). | **Ні** — cron + скрипт: скрипт читає Notion, робить те, що зараз робить твій n8n (або просто оновлює статуси / готує список). |
| **Публікація в соцмережі** | Webhook або по розкладу | Взяти пост з Notion → опублікувати в TG / LinkedIn / FB / Instagram. | **Можна лишити в n8n** — у тебе вже є воркфлоу; бот/скрипт тільки готує дані в Notion або викликає webhook з «опублікуй запис X». |
| **SEO в статтю/кейс** | — | Твій готовий воркфлоу в n8n. | **Залишається в n8n** — нічого не міняємо. |
| **Новини: моніторинг RSS** | Cron раз на 6 год | Забрати RSS → фільтр через Claude (оцінка 7+/10) → кандидати в Notion або в TG на затвердження. | **Ні** — окремий скрипт по cron: RSS → Claude → запис у Notion або повідомлення в бота. |
| **Рерайт новини** | Після затвердження новини | Взяти текст → Claude (news_rewrite) → записати в Контент-план. | **Ні** — скрипт або бот: виклик Claude, запис у Notion. |
| **Сайт (Netlify rebuild)** | Оновлення контенту | Після змін у Notion викликати rebuild сайту. | **Ні** — скрипт: виклик Netlify build hook. Або вже є в n8n — можна лишити один міні-воркфлоу. |
| **Картинки (Replicate), R2, візуали кейсів** | За запитом / по події | Генерація зображень, завантаження в R2, оновлення полей в Notion. | За бажанням — або скрипт (Replicate API, R2 S3-совместимий), або один n8n воркфлоу, якщо тобі так зручніше. |

Підсумок: **жоден із цих процесів не вимагає саме n8n.** Всі можна зробити ботом + скриптами + (за потреби) одним–двома існуючими воркфлоу в n8n.

## Що реалізувати без n8n

- **Telegram-бот** (один процес/додаток):  
  - прийом повідомлень і callback від кнопок;  
  - команди: /strategy, /plan, /today, /news, /drafts, /stats, /pause, /resume, /newpost, /regenerate, /prompt, /r2stats;  
  - всередині: запити до Notion, виклики скриптів або HTTP (Claude, Replicate, R2 — якщо потрібно), відправка відповідей у Telegram.

- **Скрипти (Python), які бот викликає або які крутяться по cron:**  
  - strategy_runner: промпти #1–#4 → Claude → запис у Notion;  
  - plan_generator: згенерувати план на 30 днів → Notion (і опційно повідомити в TG);  
  - content_scheduler: «що сьогодні» з Notion, оновлення статусів або підготовка списку на публікацію;  
  - news: RSS → Claude (фільтр) → Notion або TG; рерайт новини → Notion;  
  - при потребі: Replicate + R2, Netlify build hook — теж скрипти.

- **n8n мінімум (якщо хочеш лишити):**  
  - твої вже готові воркфлоу (наприклад, SEO для статті/кейсу);  
  - за бажанням — один воркфлоу «опублікувати з Notion в соцмережі» (webhook: «опублікуй запис X»), решта логіки — в боті/скриптах.

Тобто: **процес, який я хотів би реалізувати** — це саме це: один бот + набір скриптів; n8n тільки там, де ти сам хочеш його використовувати (наприклад, публікація або SEO), без обов’язкових 16 воркфлоу з ТЗ.

Якщо ок — далі можу запропонувати структуру папок і конкретні кроки: спочатку бот (команди /plan, /today, читання з Notion), потім strategy_runner скрипт і підключення його до /strategy.
