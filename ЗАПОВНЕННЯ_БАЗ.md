# Як прибрати сміття, заповнити всі бази і проставити дати в графіку

Один чекліст: спочатку залишаємо одну набір баз, потім заповнюємо їх контентом, потім проставляємо дати в Контент-плані.

---

## Крок 1. Залишити лише потрібні бази (прибрати дублікати)

У Notion на батьківській сторінці зараз може бути кілька баз з однаковою назвою (дублікати). Залишити треба **по одній** базі на кожну назву; ID цих баз мають збігатися з тим, що в `NOTION_DATABASE_IDS.json`.

**Що зробити:**
```bash
python scripts/notion_remove_duplicates.py
```
Скрипт перемістить зайві бази в кошик; залишить ту, ID якої в JSON (або першу з однаковою назвою).

**Якщо є бази з різними назвами** (наприклад «Контент-план» і «📋 Контент-план») — дублікат по назві не зникне (різні назви). Тоді вручну видали в кошик ту базу, якою не користуєшся, і переконайся, що в `NOTION_DATABASE_IDS.json` вказано ID саме тієї бази, з якою працюють скрипти.

---

## Крок 2. Які бази є і хто їх заповнює

| База (назва в TZ) | Що заповнює | Скрипт |
|-------------------|-------------|--------|
| Контент-кути | 7 кутів (хук, інсайт, факт, твердження) | `import_initial_angles.py` |
| Контент-план | 7 заглушок по буферних темах (без дат) | `seed_bases_initial.py` (потім дати — крок 4) |
| Стратегія/ICP | 1 сторінка (проблема + ICP) | `seed_bases_initial.py` |
| Бібліотека промптів | 12 промптів | `fill_prompts_notion.py` |
| Бібліотека книг | 50 книг (назва + автор) | `seed_bases_initial.py` |
| Джерела новин | 3 RSS-джерела | `seed_bases_initial.py` |
| Конкуренти | — | порожня, заповнюється вручну або окремим скриптом |
| Статистика | — | порожня |
| Медіа-бібліотека | — | порожня |

Контент-план також має поле **Буферна тема** і сторінку **Графік публікацій** — їх додає `setup_schedule_and_buffer.py`.

---

## Крок 3. Порядок запуску скриптів (щоб все заповнилось)

Запускай **з кореня проекту** (де лежить `.env` і `NOTION_DATABASE_IDS.json`):

1. **Поле «Буферна тема» і сторінка «Графік публікацій»** (якщо ще не робив):
   ```bash
   python scripts/setup_schedule_and_buffer.py
   ```

2. **Промпти** (12 штук):
   ```bash
   python scripts/fill_prompts_notion.py
   ```

3. **Контент-кути** (7 кутів):
   ```bash
   python scripts/import_initial_angles.py
   ```

4. **Стратегія, книги, джерела новин, 7 заглушок у Контент-план** (Gemini, при 429 — пауза і повтор):
   ```bash
   python scripts/seed_bases_initial.py
   ```

Після цього заповнені: Контент-кути, Контент-план (7 записів без дат), Стратегія, Книги, Джерела новин, Промпти. Конкуренти / Статистика / Медіа — порожні, за потреби заповнюються окремо.

---

## Крок 4. Проставити дати в графіку (Контент-план)

Записи в Контент-плані після `seed_bases_initial` мають буферну тему, але **без дати**. Щоб графік (календар) мав дати:

```bash
python scripts/fill_content_plan_dates.py
```
За замовчуванням скрипт бере всі записи **без дати**, сортує по даті створення і присвоює їм дні **з завтра** по черзі (перший — завтра, другий — післязавтра тощо), не більше 31 запису.

**Почати з конкретної дати:**
```bash
python scripts/fill_content_plan_dates.py --start 2026-03-03
```

**Обмежити кількість записів:**
```bash
python scripts/fill_content_plan_dates.py --max 7
```

Після цього у вигляді **Calendar** (по полю «Дата») буде видно графік публікацій з датами.

---

## Підсумок: мінімум для «все заповнено і графік з датами»

1. `notion_remove_duplicates.py` — прибрати дублікати баз.  
2. `setup_schedule_and_buffer.py` — буферна тема + сторінка Графік.  
3. `fill_prompts_notion.py` — промпти.  
4. `import_initial_angles.py` — кути.  
5. `seed_bases_initial.py` — стратегія, книги, джерела, 7 записів у Контент-план.  
6. `fill_content_plan_dates.py` — проставити дати в Контент-плані.

У Notion: відкрити базу **Контент-план** → додати вигляд **Calendar** → Property = **Дата**. Тоді графік виглядає як календар з датами.
