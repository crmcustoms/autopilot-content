# Джерела новин: підбір, статус, логіка перепису

## Статус і не видаляти

- У базі **Джерела новин** у кожного запису є поле **Статус**: **Активний** / **Неактивний**.
- Джерела **не видаляють** — непотрібні роблять **Неактивними** в Notion. Так зберігається історія і можна знову вмикати джерело.

Будь-який скрипт або воркфлоу, який бере список джерел для обробки, має фільтрувати лише записи зі **Статус = Активний**.

---

## Підбір джерел: офіційний сайт і блог Planfix

**Основний запит:** брати новини з офіційного сайту та блогу planfix.ru (і при потребі planfix.com), переписувати і публікувати від імені компанії.

Додати джерела Planfix в Notion:

```bash
python scripts/add_planfix_sources.py
```

Скрипт додає (якщо таких URL ще немає):

| Назва | RSS URL |
|-------|--------|
| Planfix — оф. блог (planfix.ru) | https://planfix.ru/blog/feed/ |
| Planfix — блог (planfix.com) | https://planfix.com/blog/feed/ |

Інші джерела (DOU, Mind.ua тощо) залишаються в базі; їх можна зробити **Неактивними**, якщо не використовуєш.

---

## Логіка публікації: не як своє, а як новина з оф. джерела

Публікувати треба **не як власний контент**, а як повідомлення про те, що на офіційному сайті/блозі Planfix з’явилась така новина, огляд або кейс.

Приклад формулювання:

- «На офіційному блозі Planfix вийшла новина: …»
- «У Planfix опублікували огляд / кейс: … Деталі на planfix.ru/blog/…»

Тобто: **взяти статтю з RSS → переписати коротко (або адаптувати) → опублікувати з чіткою посиланням на джерело** і формулюванням на кшталт «на оф. сайті/блозі Planfix вийшла така новина/огляд/кейс».

Промпт для перепису (у воркфлоу або боті) варто включати цю рамку, щоб модель не видавала текст за власний контент компанії.
