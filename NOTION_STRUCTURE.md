# Структура Notion: календар і теми

**Щоб прибрати сміття, заповнити всі бази контентом і проставити дати в графіку** — покрокова інструкція в **ЗАПОВНЕННЯ_БАЗ.md**.

Одна база **Контент-план** — це і календар публікацій, і пул постів по темах. Дублікатів бути не має.

---

## 1. Видалити дублікати

Запусти: `python scripts/notion_remove_duplicates.py`

Скрипт перемістить зайві бази з однаковою назвою в кошик; залишить по одній на кожну назву (пріоритет — ID з NOTION_DATABASE_IDS.json).

---

## 2. Як зробити календар справжнім календарем

1. Відкрий базу **Контент-план** (ту, що залишилась після видалення дублікатів).
2. Додай вигляд: **+ Add a view** → **Calendar**.
3. У налаштуваннях виду вкажи **Date property** = **Дата**. Збережи.
4. Щоб бачити пости по темах — додай ще один вигляд **Board**, **Group by** = **Буферна тема** (або **Тема**).

Тоді календар виглядатиме як календар (дні/тижні), по кліку — пости; Board покаже теми з посиланнями на тексти.

**Перелінковка:** у Контент-плані є поле **Кут** (relation на Контент-кути). Відкриваєш запис — бачиш текст поста і пов’язаний кут. Окремих «баз по темах» не потрібно: тема = значення **Буферна тема** в цій же базі; Board по цьому полю = розбивка по темах.

---

## 3. Логіка «щоб у кожної теми вистачало контенту»

Це робить скрипт/бот:
- Рахує по базі Контент-план: скільки записів зі статусом не «Опубліковано» по кожній **Буферна тема**.
- Якщо по темі менше 2 — генерує (наприклад, 7 нових постів) і додає їх у ту ж базу.

Окремих баз-буферів на кожну тему не створюємо — одна база, вигляди Calendar і Board.
